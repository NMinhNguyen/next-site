First, letâ€™s talk about how Next.js handles **static assets** such as images.

Next.js can serve static files, like images, under **the top-level `public` directory**. Files inside `public` can be referenced using the base URL (`/`).

If you take a look at our Next.js app, inside `<footer>`, we refer to the image at `/zeit.svg`:

```jsx
Powered by <img src="/zeit.svg" alt="ZEIT Logo" />
```

You can check that `zeit.svg` exists inside the `public` directory.

The `public` directory is also useful for `robots.txt`, Google Site Verification, and any other static assets. Check out <Link href="/docs/[...slug]" as="/docs/basic-features/static-file-serving"><a>our documentation</a></Link> to learn more.
